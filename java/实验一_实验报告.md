# 实验一 Spring Boot 整合 MyBatis-Plus 的搭建实验报告

## 一、实验课程名称
Java 高级应用开发

## 二、实验项目名称
搭建 Spring Boot + MyBatis-Plus 后台，并接入 DeepSeek AI 接口

## 三、实验类型
设计型

## 四、实验目的与要求
1. 理解 Spring Boot 项目的基本结构，掌握启动类、依赖管理、自动配置等核心内容。
2. 掌握三层架构的基本写法，即 `Controller`、`Service`、`Mapper` 和实体类 `Pojo/Entity` 的组织方式。
3. 掌握 MyBatis-Plus 的使用方法，并使用其自带的增删改查能力完成数据库操作。
4. 掌握多环境配置文件的写法，能够通过 `application.yml`、`application-dev.yml`、`application-prod.yml`、`application-min.yml` 管理不同环境。
5. 掌握 Restful API 风格接口的编写方法。
6. 在 Spring Boot 项目中接入 AI API，本实验接入的是 DeepSeek，并通过 Spring AI 完成调用。

## 五、实验原理与内容
本实验基于 Spring Boot 搭建后端服务，采用三层架构组织代码：
1. `Controller` 层负责接收前端请求并返回结果。
2. `Service` 层负责业务逻辑处理。
3. `Mapper` 层负责通过 MyBatis-Plus 访问数据库。
4. `Entity` 用于封装数据库表字段。

实验中一共完成了以下内容：
1. 使用 Maven 管理依赖，引入 Spring Boot、MyBatis-Plus、MySQL、Lombok、Spring AI、DeepSeek 等依赖。
2. 配置 MySQL 数据源，并使用 MyBatis-Plus 自带的 `list`、`getById`、`save`、`updateById`、`removeById` 完成增删改查。
3. 创建 `UserController`、`UserService`、`UserServiceImpl`、`UserMapper`、`User` 实体类，完成三层架构。
4. 配置多环境文件，在开发环境中配置数据库和 DeepSeek，在最小环境中关闭 AI，避免测试启动报错。
5. 使用 Spring AI 接入 DeepSeek，编写 `/api/ai/generate` 接口。
6. 将前端页面 `index.html` 放入静态资源目录，通过页面输入问题并调用 DeepSeek 接口返回结果。

## 八、操作异常问题与解决方案
1. 问题：原项目使用的是 Spring Boot 4.0.3，而 Spring AI 当前版本主要适配 Spring Boot 3.4.x/3.5.x。  
   解决方案：将 `pom.xml` 中的 Spring Boot 版本调整为 `3.4.4`，并将 MyBatis-Plus starter 改为 `mybatis-plus-spring-boot3-starter`。

2. 问题：AI 接口需要配置 DeepSeek API Key，如果未配置就会导致调用失败。  
   解决方案：在 `application-dev.yml` 和 `application-prod.yml` 中通过 `${DEEPSEEK_API_KEY:}` 读取环境变量；同时在 `AiServiceImpl` 中增加空配置提示，避免程序直接报错。

3. 问题：测试或最小环境不一定有 AI Key，容易导致启动或测试失败。  
   解决方案：在 `application-min.yml` 中关闭 DeepSeek Chat，使最小环境只保留基础功能。

4. 问题：多环境接口路径不一致可能导致前端请求失败。  
   解决方案：在开发环境中统一使用 `/api` 作为上下文路径，并在前端页面中固定请求 `/api/ai/generate`。
